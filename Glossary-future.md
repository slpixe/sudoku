# Glossary Future

This is a parking lot for terms that are not part of the agreed active vocabulary yet. Move terms into `Glossary.md` only after we decide they are part of the product or collaboration language.

## Auto Notes

Possible future term for generated notes shown by the app.

Current state: there is internal code and translated text related to generated notes, but no visible control currently exposes this as a main UI feature.

Related code words: `showHints`, `notePossibilities`, `autoNotes`, `show_auto_notes`.

## Candidates

Possible technical term for numbers that could legally fit in a cell.

Use if solver/generator discussions need a precise word for possible values. Avoid using this for player-entered Notes unless we intentionally separate solver language from player language.

## Settings

Possible future term for a user-facing area that exposes preferences such as Clash, Occurrences, Matching Numbers, or theme.

Current state: translations include Settings-related text, but the current active controls are action buttons rather than a full settings surface.

## Shortcuts Panel

Possible future term for a visible keyboard shortcuts/help UI.

Current state: keyboard shortcuts exist, including notes copy/paste, but the shortcuts copy does not appear to be rendered in the current active UI.

## Theme Toggle

Possible future term for the dark/light mode control in the top bar.

Current code words: `DarkModeButton`.

## Solution

Possible future term for the completed answer grid. Keep separate from Hint Answer, which is one answer inserted into one selected cell.

Current code words: `solution`, `solvedSudoku`.

## Share URL

Possible future term for the route/search parameters that encode a selected game.

Current code words: `sudokuIndex`, `sudoku`, `sudokuCollectionName`, `useGameRouteSync`.

## Puzzle Generator

Possible future term for the scripts and engine logic that create puzzle data.

Current code words: `generate`, `generate_sudokus.ts`.

## Solver

Possible future term for the engine logic that solves puzzle data and validates generated puzzles.

Current code words: `solverAC3`, `solverBacktracking`, `solve`.
